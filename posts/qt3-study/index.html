Making an Application Debuggable The release version of an application doesn’t contain any debugging symbols or other debugging information. During development it is useful to produce a debugging version of the application that has the relevant information. This is easily achieved by adding debug to the CONFIG variable in the project file.

For example:

CONFIG += qt debug 
HEADERS += hello.h 
SOURCES += hello.cpp 
SOURCES += main.cpp

Use qmake as before to generate a Makefile and you will be able to obtain useful information about your application when running it in a debugging environment.
